표시 규칙 (Format Rules)

  1. 잔여보장부채(LRC)·발생사고부채(LIC) 분리 표기 — 각 상품군마다 LRC, LIC 2개 열로 구분.
  2. LRC = 손실요소외 + 손실요소 합산 — Loss Component를 별도 열로 분리하지 않고 LRC 한 열에 합산. 손실요소 인식·환입은 Ⅰ-2-4 행으로 표시되지만 열은 LRC에 귀속.
  3. 상품군별 분해 — 사망보장 · 건강보장 · 저축성 · 변액 · 기타 (생보 표준 5분류). 각 상품군은 색상으로 구분.
  4. 잔액 표시 순서 — 기초잔액은 항상 표 최상단, 기말잔액은 항상 표 최하단.

주석

  1. DART XBRL 역할 DI817100 (IFRS17 §104 LRC/LIC 변동분 차이조정)의 상품별 구분 sub-table. 발행보험계약, 별도(SeparateMember) 기준.
  2. 잔액 검증: 기말 잔액 합계 269,967억원 ≈ 27.00조원 = 미래에셋생명 제39기 BS 보험계약부채와 일치 ✓
  3. 상품군 매핑 (pre.xml에서 확정): 사망=dart_LifeInsuranceMember, 건강=dart_HealthInsuranceMember, 연금=entity Pension...IDeathTable, 저축=entity Savings...IDeathTable, 기타=dart_OtherInsuranceMember.
  4. 각 라인은 LRC 또는 LIC 한 쪽에 자연 귀속 — 손실부담계약(환입)만 LRC(−)·LIC(+) 양쪽 분해, 잔액 행도 분해. 그 외 라인은 한쪽 col에만 값.
  5. 보험수익·투자요소·OCI 일부 라인은 미래에셋이 상품별이 아닌 배당여부별(유배당계약/유배당 외 계약)로만 공시 → 5상품군 cell은 빈 칸, 합계 열에 (유배당+유배당외) 합산값 표시. DI817100의 별도 sub-table(Table 1, 배당여부별 구분)에 동일 데이터가 분해되어 있음.
  6. 발생사고요소 조정은 INSTANT 시점(잔액성) element로 보고됨 — duration 변동 cell에는 안 나타남 (별도 처리 필요).
  7. 지급 보험금·서비스비용은 element가 ...PaidForInsuranceContractsIssuedExcludingInsuranceAcquisitionCashFlowsInsuranceContractsLiabilityAsset (취득CF 제외분).
  8. 금액 부호: 부채 증가(+), 감소(−). 합계 = 5상품군 단순 산술합. 5천만 미만은 빈 칸.
