표시 규칙 (Format Rules)

  1. 잔여보장부채(LRC)·발생사고부채(LIC) 분리 표기 — 각 상품군마다 LRC, LIC 2개 열로 구분.
  2. LRC = 손실요소외 + 손실요소 합산 — Loss Component를 별도 열로 분리하지 않고 LRC 한 열에 합산. 손실요소 인식·환입은 Ⅰ-2-4 행으로 표시되지만 열은 LRC에 귀속.
  3. 상품군별 분해 — 사망보장 · 건강보장 · 저축성 · 변액 · 기타 (생보 표준 5분류). 각 상품군은 색상으로 구분.
  4. 잔액 표시 순서 — 기초잔액은 항상 표 최상단, 기말잔액은 항상 표 최하단.

주석

  1. 본 표는 DART XBRL 역할 DI817100(IFRS17 §104 LRC/LIC 변동분 차이조정)의 발행보험계약 부분. 보유재보험은 DI817200에서 별도 공시.
  2. 재무제표 구분 = SeparateMember(별도). 분류 축(DisaggregationOfInsuranceContractsAxis) = InsuranceContractsIssuedMember 만 포함.
  3. 잔액 검증: 기말 잔액 합계 269,967억원 ≈ 27.00조원은 미래에셋생명 제39기 BS의 보험계약부채와 일치 (사망+건강+연금+저축+기타 = 발행보험 전체).
  4. 상품군 axis 멤버 (TypesOfContractsAxis):
    • 사망 = dart_LifeInsuranceMember
    • 건강 = dart_HealthInsuranceMember
    • 연금 = entity00112332_PensionInsurance...IDeathTable...
    • 저축 = entity00112332_SavingsInsurance...IDeathTable...
    • 기타 = dart_OtherInsuranceMember
  5. 각 상품군 sub-column: 잔여보장부채(LRC) = 손실요소외 + 손실요소 합산, 발생사고부채(LIC). 손실요소 인식·환입(1-4)은 LRC 열에 귀속.
  6. 일부 변동 element는 LRC/LIC 분해 없이 보고됨(예: 보험수익·기타변동·지급보험금) → 빈 칸 처리. 사업보고서의 21-1 표와 모든 라인이 1:1 매칭되지는 않음.
  7. 금액 부호: 부채 증가(+), 감소(−). 합계 = LRC + LIC (대수합). 5천만원 미만은 빈 칸.
